Module 2 · Core Concept

What is an agent?

An agent is a system that perceives its environment, makes decisions, and takes actions to achieve a goal — autonomously.

Perceive
Read the environment
Scans your codebase, reads files, captures error messages and context.
Decide
Formulate a plan
The language model reasons about what to do next and in what order.
Act
Take real actions
Edits files, runs commands, fixes errors, commits code — without you.
Regular LLM (chat)
Answers questions
Responds to one prompt at a time
No access to your files or terminal
You do all the copy/paste work
Forgets everything next session
Agent (Claude Code)
Completes tasks
Plans and executes multi-step tasks
Reads, writes, and runs files directly
Acts autonomously until task is done
Remembers rules via CLAUDE.md
Key insight
An agent is not just a smarter chatbot. It is a system that takes action in your environment — not just text in a conversation.
Module 2 · How It Works

The agent loop

Every agent runs the same cycle — over and over until the task is complete.

1
You describe the goal, not the steps
"Find and fix the bug" — not "open file X, look at line Y, change Z".
2
The agent figures out the steps itself
Reads the codebase, identifies the problem, writes a fix, runs the code, checks the result.
3
It loops until the task is done
If the first fix fails, it observes the new error and tries again — autonomously.

What Claude Code can do
✓
Read any file in the project
Scans folder structure and source files to understand the full codebase.
✓
Write and edit files directly
Creates and edits code — VS Code shows you the diff before you accept.
✓
Run scripts and fix errors
Executes python src/ner_pipeline.py, reads output, iterates until it works.
✓
Remember rules via CLAUDE.md
Entity schema, coding conventions, folder rules — loaded every session automatically.
Important
Claude Code only touches files in the folder you open it in. It always asks permission before significant changes — you stay in control.

Module 2 · Tools

How tool use works

The model cannot read files on its own — tools bridge the gap between language and the real environment.

The model is given plain text instructions describing available tools and how to call them.
When it wants a tool, it responds with a structured call like ReadFile: main.go.
The coding assistant intercepts that, runs the tool, and feeds the result back.
The model uses the result to formulate the final answer.
Key idea
The model never touches files. It only speaks text — the assistant runs the tool.
1 · You ask
2 · Tool call
3 · Tool runs
4 · Answer
You send the task to the assistant
Module 2 · Memory

How Claude remembers — CLAUDE.md

Claude Code has no built-in memory between sessions. Instead it reads a markdown file called CLAUDE.md at startup — this is its memory.

What goes in CLAUDE.md
# Project rules
- Use Python 3.11+
- Follow PEP 8 style
- Never modify raw data/

# NER entity schema
PER, ORG, LOC, DATE, MED

# Preferred libraries
spaCy for NLP tasks
pytest for testing

# Output format
Always return JSON
Key idea
Whatever you put in CLAUDE.md is loaded into the context window automatically every session — no need to repeat instructions.
Where memory fits in the system
Three types of CLAUDE.md
Where you place it matters
Project root (/project/CLAUDE.md) — shared via git. Local (.claude/CLAUDE.md) — personal. Global (~/.claude/CLAUDE.md) — every project.
Auto-loaded every session
No setup needed
When you run claude in a project folder, it automatically finds and loads CLAUDE.md. The model sees your rules before your first message.
Module 2 · Memory

Conversation history

Every conversation with Claude Code is stored locally on your machine — you can clear it, compact it, or resume an older one.

Three commands to know
↻
/compact — trim the context
Keeps a summary of what happened but frees up context window space. Use mid-session when the conversation gets long.
✕
/clear — full reset
Wipes the entire conversation history. Claude starts fresh with only CLAUDE.md. Use when starting a completely new task.
⟳
Resume — continue previous session
If you closed Claude Code mid-task, you can resume and pick up exactly where you left off.
How memory and history work together
How to use this in practice
Start a session → work with Claude → use /compact when the context gets long → use /clear when starting a completely new task. Your CLAUDE.md rules always persist across all of these.
